        In total, the client can process 6 types of requests.

        SendClientCheck requests
        Type Description Current client version
        2 or 0x2 This type returns 32 flags from the CPhysicalSA structure . If the player is in a vehicle, then it returns information about the vehicle, and if on foot, then information about the player himself. These are flags such as whether the player is standing on the ground, whether he is in the water, whether he touches the water, whether he is vulnerable to various types of damage <= 0.3z or >= 0.3.7 R2, 0.3.7 R3, 0.3DL-R1
        5 or 0x5 It constructs a checksum of size in bytes from the address arg + offset (GTA memory) and returns it in the response argument. Valid addresses: 0x400000 - 0x856E00. Valid offsets: from 0 to 255. One of the most interesting and useful types. You can check if the memory is packed with some kind of cheat/hack. < 0.3z or >= 0.3.7 R2, 0.3.7 R3, 0.3DL-R1
        69 or 0x45 It constructs a checksum of size in bytes from the address arg + offset (SAMP memory) and returns it in the response argument. Valid addresses: 0x0 - 0xC3500. Valid offsets: 0 - 255. Can also be used to search for installed cheats/hacks. >= 0.3.7 R2, 0.3.7 R3, 0.3DL-R1
        70 or 0x46 This type reads the data of the CBaseModelInfoSA structure of the specified model and returns a one-byte checksum. You can specify which offset to start reading from and how many bytes to read. For example, the SendClientCheck(playerid, 0x46, 1598, 0, 28) request will return the checksum of the first 28 bytes of the 1598 model structure. <=0.3z or >= 0.3.7 R2, 0.3.7 R3, 0.3DL-R1
        71 or 0x47 This type reads the data of the CColModelSA structure of the specified model (the pointer to which lies at offset 20 in the CBaseModelInfoSA structure ), in particular the checksum of the collision. <=0.3z or >= 0.3.7 R2, 0.3.7 R3, 0.3DL-R1
        72 or 0x48 This request does not require any additional arguments. It returns the number of seconds since the computer was started in arg. <=0.3z or >= 0.3.7 R2, 0.3.7 R3, 0.3DL-R1

        Clue

        		 The check amount is calculated according to a simple algorithm:
        for(new i = startaddress; i < endaddress; i++) result ^= ReadMemory(i) & 0xCC;
